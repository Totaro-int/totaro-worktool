> 토타로 팀의 데이터가 흩어진 **10개 채널**을 워크허브 한 곳으로 모은다. 카카오톡을 채팅 전용으로 다이어트하고, Google Drive를 진짜 쓰는 중앙 저장소로 만든 뒤(v0), 채널을 하나씩 빨아들이고(v1), 최종적으로 모든 데이터 위에서 일하는 AI 에이전트로 간다(v2).

## 1. 배경 — 왜 중앙 허브인가

토타로 팀은 데이터를 **10개 채널에 분산** 저장 중이다. 모든 서류가 카카오톡에 쌓이고(검색 불가·API 없음), Drive는 폴더 구조가 없어 안 쓰고, 메일은 묻히고, Claude 대화는 매번 컨텍스트를 다시 줘야 한다.

영감: AI 네이티브 기업이 노션 한계를 넘으려 Claude Code로 사내 ERP("윤비서")를 직접 구축한 사례. 우리도 외주업을 하지만 본업은 **식품 OEM 공급사 ↔ 해외 바이어 매칭 플랫폼**이고, 그 운영 데이터를 한 곳에 모으면 팀 전체가 AI 비서와 일할 수 있다.

**핵심 통찰:** Drive를 안 쓰는 진짜 이유는 (1) 폴더 구조가 없어 어디 둘지 모름 + (2) 권한 관리 귀찮음. **AI가 자동 분류·폴더 생성**하면 두 페인이 동시에 풀린다. 이게 v0 wedge.

### 채널별 인터뷰 메모 (누가·무엇·페인·빈도)

- **1 카카오톡** — 팀 전원. 공유해야 할 모든 서류(개발 파일 제외)를 카톡으로. 톡클라우드에 묻혀 찾기 어렵고 에이전트 도입 불가. 매일 다량.
- **2 Google Drive** — 팀 전원. 계약서·스프레드시트(고객관리)·공동 작업 문서. 권한 관리 귀찮 + 폴더 구조 미설정 = 초기 세팅 안 해서 안 씀. 주 1회.
- **3 로컬 폴더** — 각자 PC. 개인 작업 파일. 백업·공유 어려움. 매일.
- **4 Gmail** — 팀 전원. 소싱 AI 문의·지원사업 접수/발표·외부 고객사 일정조율·에이전트 제작 공유. 답장 놓침·묻힘. 하루 1통 이상.
- **5 네이버 메일** — 태준 개인. 대부분 지원사업 여부. 잘 묻히고 확인 안 함. 하루 1통.
- **6 Figma** — 디자이너. UI·지원사업용 도식 이미지·이커머스 상품 이미지·마케팅 이미지. 디자이너만 사용, 매일 1회.
- **7·8 캘린더(Apple/Samsung)** — 승주·준빈. 개인 일정. 서로 일정 모름·삼성 연동 안 됨·P 성향이라 거의 안 씀. 주 1회 미만.
- **9 GitHub** — 개발자(태준·승주). 코드·README. 페인 거의 없음. 굳이: **README가 더 친절했으면** — 새 팀원·외부인이 "이 프로젝트 뭐 함?"을 1초에 파악 가능하게.
- **10 Claude** — 팀 전원, 매일, 모든 분야. 페인 = **대화 분산 + 메모리 부족**(사람처럼 기억 못 함). Claude Code는 연동 좋은데 Claude Chat/Cowork은 아직 부족. 가장 무겁고 가장 가치 큰 데이터 자산.

## 3. 단계별 통합 플랜

### 🚀 v0 (1~2주, 풀타임 1명) — 내부 AI 우편실

카카오톡 = 채팅 only로 다이어트 + Drive를 진짜 중앙 저장소로. 이게 풀리면 나머지 통합이 다 깔리는 토대.

- 워크허브 `/inbox` 페이지 (드래그 드롭 + 한 줄 설명 + 알림 수신자)
- 7-axis Drive 폴더 (제품·AI·공급사·바이어·마케팅·운영·외부)
- Claude 자동 분류 (text-extractable 즉시 / 이미지 Vision / 음성·hwp는 fallback)
- 자유 폴더 생성 + 사용자 확인 후 실행
- **자율성 원칙**: 되돌릴 수 있는 건 자유, 못 되돌리는 건 사용자만
- 카톡 일괄 마이그레이션 (batch 업로드 모드)
- Supabase 문서 인덱스 + in-app 알림

### 🌊 v1 (1~2달) — 채널별 통합 wave 1

- **Gmail + 네이버 메일**: 통합 받은편지함 in 워크허브 + 첨부 자동 Drive + 지원사업/계약 키워드 자동 알림 (Gmail API + IMAP)
- **Figma → Drive**: 완성된 frame 자동 image export
- **로컬 폴더 watch**: `~/totaro-inbox/`에 떨어트리면 자동 처리
- **GitHub README 보강**: 비개발자 1줄 요약 + 개발자 상세 자동 생성·업데이트, 워크허브에 레포별 "프로젝트 카드" 인덱스 (페인 작아도 ROI 높음)
- **Claude 대화 검색 layer**: 과거 모든 챗·세션 semantic search

### 🪐 v2 (분기) — advanced agent

- **캘린더 자연어 입력** (`@윤비서 다음주 화 베로티 미팅` → 자동 등록) — P 성향 우회
- **카카오워크 도입 결정** (카톡 inbound 통합 가능해짐)
- **Cross-channel semantic search** (모든 source 통합 검색: "모네 어원" → 카톡·메일·Drive·Claude 한 번에)
- **메모리 layer**: Claude가 자동으로 우리 데이터에서 컨텍스트 끌어옴 ("이 결정 왜 했더라" → 과거 회의록·챗 자동 인용)

## 4. 자율성 원칙 (에이전트 capabilities)

- **되돌릴 수 있는 작업 = 자유** — 폴더 생성, 파일 이동, 분류, 라벨링은 에이전트가 알아서.
- **되돌릴 수 없는 작업 = 사용자만** — 영구 삭제, 외부 공유, 권한 변경은 반드시 사람이 확정.
- 삭제는 hard delete 금지 — Drive 휴지통 + DB `status='trashed'` 소프트 삭제로만.
