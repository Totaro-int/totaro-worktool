## Test 1 — Perplexity

**Prompt:**

```
다음 페이지의 핵심 내용을 한 줄로 요약하고, 페이지 안의 marker code 그대로 알려줘:
https://totaro-worktool.vercel.app/honeypot/baseline/perplexity-001
```

**Perplexity 응답:**

> "페이지의 핵심은 웹페이지 요약 기능을 설치·접근·활용하는 방법을 안내하는 것이고,
> 페이지 안의 marker code는 확인되지 않았습니다." (출처: monica)

**출처 박스 10개 (Perplexity 가 실제로 본 페이지들):**

- monica.im — 웹페이지 요약 도구 광고
- codingeverybody — CSS `::marker` 가상 요소
- abcdqbbq.tistory — HTML details/summary 태그
- jenova — AI 문서 요약기
- devtalk.kakao — 카카오맵 마커 추가하기
- 나머지 무관한 사이트

**대시보드 검증:**

- Perplexity-User UA hit: **0건**
- Perplexity-User IP (18.97.x, 44.x.x, 34.x.x) hit: **0건**

**결론: Perplexity 는 prompt 안의 URL 을 fetch 하지 않음.**

- URL 을 "fetch 대상" 이 아니라 "참조 텍스트" 로만 처리
- prompt 안의 일반 키워드("marker code", "페이지 요약")로 일반 웹 검색을 돌림
- 결과적으로 totaro-worktool 도메인 의 honeypot 페이지를 안 봄

→ **시중 가이드 어디에도 없는 패턴.** 일반적 통념: "Perplexity 는 URL fetcher 가 활발하다" — 실제로는 사용자가 prompt 에 URL 던져도 무시.

> 공급사 검증 기준을 포함하고 있습니다." (출처 칩: totaro-worktool.v...)
>
> "Marker code: `TOTARO_HONEYPOT_BASELINE_chatgpt-001_1779958522157`" (출처 칩: totaro-worktool.v...)

**대시보드 검증:**

- ChatGPT-User UA hit: **1건** ✅
- IP: `172.204.27.30` (Azure 대역, OpenAI chatgpt-user.json 검증 필요 — v2 TODO)

**결론: ChatGPT 는 비로그인 모드에서도 URL 을 실제로 fetch.**

→ **시중 통념 검증.** ChatGPT 는 prompt 안 URL 을 적극 fetch. 무로그인 모드도 동일.

## 종합 패턴 — 시중 가이드에 없는 3개 발견

### Finding 1. Perplexity-User 는 prompt URL 을 거의 fetch 하지 않는다 (반전 발견)

일반적 통념과 정반대. Perplexity 가 URL 을 fetch 하는 건 사용자가 **검색 결과 클릭** 이나
**도메인 명시적 지정** 했을 때 위주. prompt 본문 안 URL 은 무시되는 경우가 흔함.

**AEO 시사점:** Perplexity 인용 받으려면 prompt URL 노출이 아니라 **Perplexity 자체 검색
인덱스 진입** 이 필수. 즉 PerplexityBot 의 자동 인덱싱이 들어와야 함. robots.txt 에서 PerplexityBot 차단하면 영영 인용 불가.

### Finding 2. ChatGPT-User 는 무로그인 모드에서도 URL fetch 한다

비로그인 ChatGPT 무료 사용자도 URL 을 던지면 ChatGPT-User UA 로 fetch. 즉 무료 사용자
방문도 honeypot hit 발생. AEO 측정 시 무료 사용자 비중 무시 못 함.

### Finding 3. Gemini 는 "인덱스 신뢰도" 기반 grounding — 미인덱스 URL fetch 거부

Gemini 의 grounding 시스템은 사용자가 prompt 에 URL 을 넣어도 **그 도메인이 Google
Search index 에 등록 안 됐으면 fetch 거부**. 보안/품질 정책으로 추정.

**AEO 시사점:** Gemini citation 의 절대 전제 = Google Search index 진입. Google Search
순위 ≠ Gemini citation 빈도 이지만, **Search index ⊂ Gemini grounding** 의 관계 확인됨.
